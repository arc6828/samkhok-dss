# คู่มือและสรุปการพัฒนาระบบ: Sam Khok DSS

ยินดีด้วย! ระบบสนับสนุนการตัดสินใจจัดการอุทกภัยอัจฉริยะ (Decision Support System: DSS) พื้นที่เทศบาลตำบลสามโคก จังหวัดปทุมธานี ได้รับการออกแบบ พัฒนา และคอมไพล์สำเร็จอย่างสมบูรณ์แบบแล้ว

---

## 🚀 ฟังก์ชันหลักที่เสร็จสมบูรณ์ (Key Accomplished Features)

ระบบได้รับการพัฒนาภายใต้กรอบการทำงาน Next.js 16 (React 19) และจัดโครงสร้างการแสดงผลแบบ **Command Center Premium Slate Theme** โดยแบ่งการเข้าถึงออกเป็น 6 ส่วนสำคัญ:

### 1. แดชบอร์ดภาพรวมอัจฉริยะ (`/dashboard`)
- **Interactive Water Level Slider**: สไลเดอร์จำลองระดับน้ำโต้ตอบแบบเรียลไทม์ (1.0 - 3.5 เมตร) ซึ่งผูกเข้ากับ State ของทั้งระบบ ทุกหน้าจะอัปเดตสถานะและระดับภัยทันทีเมื่อผู้ใช้เลื่อนแถบจำลอง
- **KPIs Indicators**: ตัวเลขแสดงยอดกลุ่มเปราะบางที่กำลังเผชิญหน้าอุทกภัยวิกฤต, จำนวนโบราณสถานที่กำลังจมน้ำ และจำนวนไร่นาแปลงข้าวที่ต้องเร่งขนย้าย
- **Recharts Analytics**: กราฟแนวโน้มระดับน้ำ 7 วันย้อนหลังเปรียบเทียบกับแนวเตือนภัย (Reference Lines) และกราฟแท่งเปรียบเทียบร้อยละความพร้อมการเก็บเกี่ยวข้าวของชาวนา

### 2. แผนที่ความเสี่ยงอุทกภัยเชิงพื้นที่ (`/map`)
- **KMZ Layer Loading**: เรียกใช้ปลั๊กอิน `leaflet-kmz` ใน React โหลดชั้นข้อมูลจากไฟล์จริง ได้แก่ ขอบเขตเทศบาล, สถานที่ราชการ, และอาคารสิ่งปลูกสร้าง ลงบนแผนที่ Leaflet แบบเรียลไทม์
- **Dynamic HTML Icons**: สัญลักษณ์กลุ่มเปราะบาง (👵/♿/🤰/🛏️) โบราณสถาน (🛕) และแปลงข้าว (🌾) จะเปลี่ยนสีและกะพริบเป็นสีแดง-เตือนภัย (Pulsating Red) ทันทีเมื่อระดับน้ำท่วมถึงระดับความสูงดินของสถานที่นั้นๆ
- **Sidebar Inspector Overlay**: คลิกที่จุดใดๆ บนแผนที่เพื่อแสดงผลข้อมูลประวัติ, ความจำเป็นทางการแพทย์, ช่องทางติดต่อด่วน หรือสถานะแปลงเกษตร พร้อมปุ่มเรียกเรืออพยพ / สั่งกระสอบทราย หรือประสานรถเกี่ยวข้าวได้ทันทีจากจุดเป้าหมาย

### 3. ทะเบียนกลุ่มเปราะบางและระบบจัดการข้อมูล (`/community`)
- **Vulnerability Breakdown**: แผงสถิติแยกตามหมวดหมู่ ผู้สูงอายุ, ผู้พิการ, สตรีตั้งครรภ์, และผู้ป่วยติดเตียง
- **Search & Advanced Filters**: ค้นหารายชื่อ หรือกรองข้อมูลอย่างละเอียดตามประเภทความเปราะบางและสถานะอพยพ
- **Full CRUD Support**: โมดอลฟอร์มลงทะเบียนกลุ่มเปราะบางใหม่ แก้ไขประวัติการแพทย์ สถานะช่วยเหลือ หรือลบข้อมูล โดยบันทึกลงฐานข้อมูลจำลอง LocalStorage เพื่อการสาธิตอย่างราบรื่น

### 4. ระบบจัดลำดับการช่วยเหลือเชิงบริหารนโยบาย (`/priority`)
- **Normalization Sliders**: ปรับเปลี่ยนค่าน้ำหนักความสำคัญของ 4 เกณฑ์หลัก (ระดับน้ำ, กลุ่มเปราะบาง, โบราณสถาน, มูลค่าการเกษตร)
- **Priority Scoring Algorithm**: ระบบประเมินจุดเสี่ยงภัยและคำนวณคะแนน (0 - 100) ถ่วงค่าน้ำหนักน้ำหลากและจัดเรียงอันดับความช่วยเหลือ (1, 2, 3...) แบบสด
- **Executive Dispatch Controller**: ปุ่มสั่งการหน่วย ปภ. เช่น "สั่งทีมอพยพ/เสริมคัน", "ส่งกระสอบทรายด่วน", หรือ "รถเกี่ยวข้าวฉุกเฉิน" พร้อมบันทึกลงปูมปูมสั่งการ (Command Logs Board)

### 5. ศูนย์กระจายสัญญาณเตือนภัยฉุกเฉิน (`/alert`)
- **Broadcaster Controller**: ส่งสัญญาณเตือนอุทกภัยจำลองเจาะจงกลุ่มเป้าหมาย (ทุกคนในเขตเสี่ยง, เฉพาะชาวนาข้าว, เฉพาะกลุ่มเปราะบาง) ผ่านช่องทาง SMS, Line OA, หรือหอกระจายข่าว
- **Interactive Templates**: เทมเพลตสำหรับเกษตรกรให้รีบเก็บเกี่ยว หรือประกาศด่วนให้อพยพกลุ่มเปราะบาง
- **Broadcast Simulator**: แอนิเมชันจำลองลำดับขั้นตอนการแพร่กระจายคลื่นสัญญาณ และบันทึกลงตารางประวัติส่งแจ้งข้อความ

### 6. รายงานสรุปภัยพิบัติสำหรับพิมพ์ประสารงาน (`/reports`)
- **Formal printable header**: รูปแบบรายงานสรุปสถิติเพื่อยื่นเสนอผู้บริหารระดับจังหวัดหรืออำเภอ
- **Print media CSS overrides**: ออกแบบ CSS เฉพาะที่จะซ่อนแถบเมนูข้าง (Navbar) ปรับพื้นหลังเป็นสีขาว และจัดเนื้อหาตารางให้เรียบร้อยสวยงามเมื่อคลิกพิมพ์ออกเป็นกระดาษหรือบันทึกเป็น PDF ผ่านเบราว์เซอร์ (`window.print()`)
- **Official sign-off blocks**: ลายเซ็นลงนามสำหรับเจ้าหน้าที่และผู้อำนวยการสั่งการ ปภ. ในท้ายรายงาน

## 🧪 การยืนยันความถูกต้องและการคอมไพล์ (Build Validation)

ระบบได้รับการรันคำสั่งคอมไพล์เพื่อทดสอบความเสถียรอย่างเป็นทางการ:
1. **Next.js Compilation**: ผ่านการคอมไพล์ Turbopack และ TypeScript สำเร็จทั้งหมด (Compiled Successfully)
2. **Linting Check**: ผ่านการตรวจสอบ Syntax ไม่มีข้อผิดพลาด
3. **Data Persistence**: ตัวแปรระดับน้ำและข้อมูลผู้ประสบภัยสามารถซิงโครไนซ์สถานะข้ามไปมาระหว่างหน้า Dashboard, Map, Priority, Community และ Reports ได้อย่างไร้รอยต่อผ่านระบบ Event-driven LocalStorage
