# แผนการพัฒนา: ระบบสนับสนุนการตัดสินใจจัดการอุทกภัย เทศบาลตำบลสามโคก (Sam Khok DSS)

แผนงานนี้จัดทำขึ้นเพื่อออกแบบและพัฒนาระบบสนับสนุนการตัดสินใจ (Decision Support System: DSS) สำหรับหัวหน้าหน่วยงานบรรเทาสาธารณภัยในพื้นที่เทศบาลตำบลสามโคก เพื่อการเตือนภัยอุทกภัย การจัดลำดับความช่วยเหลือ และการจัดการกลุ่มเปราะบางอย่างมีประสิทธิภาพ

---

## 📌 ความต้องการของระบบ (System Requirements)
อ้างอิงจากกรอบการวิจัย ระบบจะต้องครอบคลุม 4 ฟังก์ชันหลัก:
1. **ระบบเตือนภัยก่อนอุทกภัยสำหรับเกษตรกร (Pre-Flood Agricultural Warning)**: ช่วยเตือนภัยและจัดการลำดับการเก็บเกี่ยวข้าวให้ทันก่อนน้ำท่วม
2. **ฐานข้อมูลกลุ่มเปราะบาง (Vulnerable Citizens Database)**: ข้อมูลผู้สูงอายุ, ผู้พิการ, สตรีมีครรภ์ และสถานะการช่วยเหลือ/อพยพ
3. **ฐานข้อมูลพื้นที่กลุ่มเปราะบาง/โบราณสถาน (Historic & Sensitive Sites Database)**: ข้อมูลโบราณสถาน (เช่น วัดโบราณริมน้ำ) ความสูงของแนวคันกั้นน้ำ และสถานะการเสริมคันกระสอบทราย
4. **การจัดลำดับความช่วยเหลือ (Decision & Assistance Prioritization)**: อัลกอริทึมคำนวณคะแนนความเร่งด่วน โดยให้ผู้บริหารสามารถปรับแต่งน้ำหนัก (Weights) ของแต่ละเกณฑ์ได้แบบเรียลไทม์

## 🎨 การออกแบบรูปลักษณ์และประสบการณ์ผู้ใช้งาน (Rich Aesthetics & Premium UX)
เพื่อสร้างความประทับใจตั้งแต่แรกเห็น (Wow Factor) ระบบจะได้รับการออกแบบดังนี้:
- **Command Center Dashboard Layout**: ใช้แถบเมนูข้าง (Sidebar) สีน้ำเงินเข้ม/เทาเข้มหรูหรา และพื้นที่แสดงผลกระจกใส (Glassmorphism Effect) ร่วมกับพื้นหลังแบบไล่เฉดสี (Gradients)
- **Interactive Water Level Slider**: ผู้บริหารสามารถทดสอบปรับเลื่อน "ระดับน้ำแม่น้ำเจ้าพระยา (เมตร)" ตั้งแต่ 1.0 - 3.5 เมตร และสังเกตการตอบสนองของระบบ (สีของชุมชน, พื้นที่การเกษตร, วัดโบราณ และคะแนนความเสี่ยงที่จะเปลี่ยนเป็นสีส้ม/สีแดงทันทีในทุกๆ หน้าจอ!)
- **Dynamic Interactive SVG Map**: แผนที่จำลองแม่น้ำเจ้าพระยาไหลผ่านสามโคก มีการเคลื่อนไหวของสายน้ำ (Water Flow Animation) และมีจุดวงกลมสะท้อนระดับความเสี่ยงของแต่ละสถานที่ (เสี่ยงสูงกะพริบเป็นสีแดง-ชมพูพาสเทลหรูหรา)
- **Recharts Data Visualizations**: กราฟแสดงแนวโน้มระดับน้ำและการเก็บเกี่ยวพืชผลที่คมชัดและตอบสนองได้ทันที (Responsive Charts)

### 🗄️ 2. แฟ้มข้อมูลจำลอง (Mock Data Store)
สร้างไฟล์ข้อมูลกลาง [mockData.ts](file:///d:/nextjs/samkhok-dss/app/data/mockData.ts) เพื่อเก็บข้อมูลที่มีโครงสร้างสมจริงของสามโคก:
- **ชุมชน (Communities)**: เช่น ชุมชนวัดโบสถ์, ชุมชนเชียงรากใหญ่, ชุมชนวัดศาลาแดงเหนือ
- **กลุ่มเปราะบาง (Vulnerable Citizens)**: ข้อมูลชื่อ อายุ ประเภทกลุ่มเปราะบาง ชุมชน พิกัด และสถานะช่วยเหลือ
- **โบราณสถาน (Historic Sites)**: เช่น วัดโบสถ์, วัดสิงห์, เจดีย์ทอง ความสูงของคันกั้นน้ำ และระดับความเสี่ยง
- **พื้นที่เกษตรกร (Agricultural Fields)**: โซนเพาะปลูก พันธุ์ข้าว วันที่คาดว่าจะเก็บเกี่ยว และความเสี่ยงน้ำท่วม

## 🧪 แผนการตรวจสอบและทดสอบ (Verification Plan)

### การตรวจสอบโดยระบบอัตโนมัติ (Automated Checks)
- รันคำสั่ง `npm run build` เพื่อให้มั่นใจว่าโค้ด TypeScript และ Next.js ผ่านการคอมไพล์ทั้งหมดโดยไม่มีข้อผิดพลาด
- รันคำสั่ง `npm run lint` ตรวจสอบ Syntax ล่าสุดตามมาตรฐาน ESLint

### การทดสอบด้วยระบบทดลองจำลอง (Manual Scenarios)
1. **การจำลองสถานการณ์ระดับน้ำสูง**:
   - ไปที่หน้าแผนที่ ปรับสไลเดอร์ระดับน้ำไปที่ `3.2 เมตร`
   - ตรวจสอบว่าพื้นที่เกษตรกรรมและโบราณสถานริมน้ำเปลี่ยนสถานะเป็นสีส้ม/แดงเตือนภัยหรือไม่
2. **การจัดลำดับช่วยเหลือเชิงนโยบาย**:
   - ไปที่หน้าจัดลำดับความช่วยเหลือ ลองปรับสไลเดอร์น้ำหนัก "โบราณสถาน" เป็น 90% และ "กลุ่มเปราะบาง" เป็น 10%
   - ตรวจสอบว่าสถานที่ประวัติศาสตร์ขยับขึ้นมาอยู่อันดับ 1 ในตารางความช่วยเหลือหรือไม่
3. **การลงทะเบียนผู้ประสบภัย**:
   - ไปที่หน้าข้อมูลชุมชน ทดลองเพิ่มชื่อกลุ่มเปราะบางคนใหม่ และปรับสถานะเป็น "ได้รับการช่วยเหลือแล้ว" จากนั้นดูการอัปเดตสถิติในหน้าแดชบอร์ด
